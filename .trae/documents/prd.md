## 1. 产品概述

数据要素供需撮合桌面客户端，面向数据交易顾问，用于高效整理买方需求与卖方产品的匹配结果，加速数据要素交易撮合流程。
- 解决数据交易中供需信息不对称、匹配效率低、跟进流程分散的问题
- 目标用户：数据交易所/数据服务商的撮合顾问，提升撮合成功率与客户满意度

## 2. 核心功能

### 2.1 用户角色

| 角色 | 使用方式 | 核心权限 |
|------|----------|----------|
| 撮合顾问 | 系统登录 | 需求录入、产品筛选、匹配评分、沟通记录、报告导出 |

### 2.2 功能模块

1. **需求录入**：录入买方需求信息，包含行业、数据范围、更新频率、预算区间、合规要求等维度
2. **产品库浏览**：浏览卖方产品目录，按标签筛选，查看产品详情与合规资质
3. **匹配工作台**：为候选产品打匹配分，标记价格、交付周期、限制条件，比较多个候选方案
4. **沟通记录**：记录双方沟通纪要，设置下一步跟进日期，追踪沟通历史
5. **撮合报告**：生成推荐清单，比较多个候选方案，导出给客户查看的撮合报告

## 3. 核心流程

顾问录入买方需求 → 系统按标签自动推荐候选产品 → 顾问手动筛选与评分 → 标记价格/交付/限制 → 记录沟通纪要并设置跟进 → 生成推荐清单与撮合报告 → 导出报告给客户

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝 (#0F172A) + 琥珀色 (#F59E0B) 作为强调色，传达专业与信任感
- **辅助色**：石板灰 (#64748B) 用于边框和次要文字，翡翠绿 (#10B981) 用于成功/匹配状态
- **按钮风格**：圆角矩形，主操作使用琥珀色填充，次操作使用描边样式
- **字体**：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular
- **布局**：左侧固定导航栏 + 右侧内容区，桌面端优先，卡片式布局
- **图标**：使用 Lucide 图标库，线性风格，与整体简洁专业调性一致

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 需求录入 | 需求表单 | 白色卡片表单，标签式分组（基础信息/数据需求/预算合规），输入框、下拉选择、日期选择器 |
| 需求录入 | 需求列表 | 表格布局，状态徽章（待匹配/匹配中/已完成），搜索栏，筛选下拉 |
| 产品库浏览 | 标签筛选 | 顶部标签云，点击高亮，支持多选 |
| 产品库浏览 | 产品卡片 | 网格卡片，标签徽章、供应商名、匹配度指示条 |
| 产品库浏览 | 产品详情 | 侧滑抽屉面板，分区展示，合规认证图标 |
| 匹配工作台 | 候选产品列表 | 列表项含评分雷达图缩略、价格标签、交付周期标签 |
| 匹配工作台 | 方案对比 | 并排卡片对比，各维度评分条，差异高亮 |
| 匹配工作台 | 匹配评分 | 滑块评分组件，维度标签，总分计算 |
| 沟通记录 | 沟通纪要表单 | 对话框形式，富文本区域，日期与参与人选择 |
| 沟通记录 | 跟进设置 | 日期选择器，待办勾选列表，到期提醒标记 |
| 沟通记录 | 沟通时间线 | 垂直时间线，节点图标区分沟通类型，展开详情 |
| 撮合报告 | 推荐清单 | 排序列表，排名徽章，匹配分条形图 |
| 撮合报告 | 报告预览 | 模拟A4页面预览，报告封面与正文分页 |
| 撮合报告 | 报告导出 | 导出按钮（PDF/HTML），格式选择下拉 |

### 4.3 响应式设计

- 桌面端优先设计，最小宽度 1280px
- 内容区自适应，导航栏固定宽度 240px
- 卡片网格自适应列数
